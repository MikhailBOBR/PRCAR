# Тестирование

PRCAR покрыт несколькими уровнями проверки:

## Автоматические проверки

- `npm run lint` — кодстайл и статический анализ
- `npm run typecheck` — строгая проверка TypeScript
- `npm run test` — unit-тесты на форматирование, валидацию и permissions
- `npm run build` — контроль реальной production-сборки Next.js

## Что проверяют тесты

- форматирование цен и дат под `ru-RU`
- ограничение доступа по ролям
- валидацию регистрации и обязательность согласия на обработку ПДн
- валидацию фильтров каталога
- ограничение размера фото до 5 МБ

## Ручные сценарии из отчетов

Рекомендуется дополнительно прогонять:

- позитивный и негативный логин
- создание новой карточки автомобиля менеджером
- ошибку загрузки фото больше 5 МБ
- фильтрацию каталога по марке и цене
- добавление автомобиля в избранное
- отправку заявки на тест-драйв
- перевод заявки в `IN_PROGRESS`
- запрет доступа клиента к `/manager` и `/admin`
- смену роли пользователю через админ-панель
- отображение каталога на ширине `375px`

## Нагрузочные и интеграционные проверки

Для полного соответствия отчетам можно дополнительно использовать:

- `Postman` — API smoke tests
- `Google Lighthouse` — производительность витрины
- `OWASP ZAP` — безопасность публичных форм
- `JMeter` — нагрузочные сценарии на фильтрацию каталога и создание заявок
