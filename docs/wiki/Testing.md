# Testing

## Purpose

Automated tests are used to verify the core business logic of the project and protect the application from regressions after changes.

## Covered Areas
- validation schemas
- error helpers
- utility functions
- permission checks
- car service logic
- order service logic
- favorites logic
- registration logic
- role update logic

## Tooling
- `Vitest`

## Run Tests

```bash
npm run test
```

## Notes

The current automated tests focus on server-side logic and validation. They cover the most important business scenarios of the project.
